01 // MISSION_BRIEF

Context

Neo AI's MVP envisions a future where daily conversations are transformed into actionable insights, enhancing both productivity and self-awareness. The product pairs a discreet wearable device (Neo 1) with a companion iOS app that surfaces conversation summaries, emotional tone analysis, and extracted action items — automatically, without disrupting natural communication.

The target user: knowledge workers — executives, founders, and lawyers — whose professional value depends on retaining and acting on nuanced conversational intelligence.

02 // PROBLEM_STATEMENT

What Was Missing

FAULT_01 //
Lost Insights
Knowledge workers struggle to retain critical insights from daily conversations. Manual note-taking disrupts natural conversation flow, and key action items are consistently missed in the post-meeting haze.
FAULT_02 //
No Tone Intelligence
Emotional tone analysis is entirely absent from existing meeting tools. Professionals often realise too late that they misread the emotional register of a conversation — leading to miscommunication and missed signals.
FAULT_03 //
Fragmented Records
Existing tools produce fragmented, unstructured notes. Meetings leave knowledge workers with incomplete records, no semantic structure, and no automated extraction of commitments or follow-up items.

03 // DESIGN_OBJECTIVES

Mission Parameters

OBJECTIVE_01 //
Instant Intuition
Ensure a seamless user experience, enabling immediate app use within 5 seconds of reaching the home screen. Zero cognitive overhead at the entry point — the interface must be immediately legible and action-ready.
OBJECTIVE_02 //
Magical Activation
Create a clear "wow moment" by guiding users to their first conversation recording and insight card within the onboarding flow. The act of pairing Neo 1 should feel like activating something powerful, not configuring a device.
OBJECTIVE_03 //
Rapid Value
Deliver immediate value through accessible conversation notes and insights within three taps from the home screen. The time from "end of meeting" to "reading insights" must be frictionless.
04 // EXECUTION_LOG

Design Process

PHASE_01 //
Understanding User Needs
Conducted 15 user interviews with executives, founders, and legal professionals. Prioritised understanding two specific pain points: the need for automatic action item extraction, and the desire for emotional tone feedback — neither of which existing tools provided.
PHASE_02 //
Designing the MVP Experience
Focused on making the home screen a dynamic "conversation hub" — the latest conversation and quick actions always front and centre. The device state (active, listening, idle) became a core visual element of the home screen, not a buried settings detail.
PHASE_03 //
Actionable Item Architecture
Designed the three-tab conversation view: Highlights (AI summary + key takeaways), TODOs (extracted action items as checkable tasks), and Transcript (full searchable text). Each tab serves a different consumption mode — scanning, acting, and archiving.
PHASE_04 //
Phased Release
Rolled out the MVP in phases — core conversation recording and summary first, followed by tone analysis and the AI chat brainstorm feature. Phased release allowed the team to validate the core value proposition before layering on intelligence features.
05 // KEY_FEATURES

What We Built

FEATURE_01 //
Clean Home Screen
The interface provides immediate access to memory creation, battery status, and How-to guides — while dynamically showcasing Neo 1 device states. Users understand what's happening at a glance, without needing to navigate.
FEATURE_02 //
Conversation Intelligence
Each recorded conversation surfaces a Highlights tab with AI-generated key takeaways, a TODOs tab with auto-extracted action items as checkable tasks, and a full searchable Transcript. Three views for three different use modes.
FEATURE_03 //
Emotional Tone Tags
Conversations are tagged with detected emotional tones — Thinking, Calm, Curious, and others — surfaced at the card level on the home screen. Users can identify conversations worth revisiting based on emotional context alone.
FEATURE_04 //
Neo AI Brainstorm
A contextual AI chat interface available inside each conversation — "Brainstorm with Neo AI". Users can ask questions about the conversation content, expand on topics discussed, or generate follow-up actions directly from the transcript.
06 // MISSION_OUTCOME

Results

PILOT PROGRAM RESULTS //
NUMBERS WENT BEYOND EXPECTATIONS
The pilot successfully validated all three design objectives: users reached core functionality in under 5 seconds, the onboarding to first insight took fewer than 3 taps, and the Home Test Ride pilot produced results that exceeded every benchmark set at mission start.
