01 // MISSION_BRIEF

Context

Ather Energy's app adoption in Tier 2 and 3 cities was severely hampered by a mandatory email-based login system. In regions with lower email penetration, this single friction point was enough to stop users from ever accessing the product.

Analysis revealed a 68% abandonment rate directly attributed to the email login requirement, and social login complexity contributed to a 42% overall decrease in app adoption in these regions. The fix wasn't more onboarding — it was removing the barrier entirely.

02 // PROBLEM_STATEMENT

What Was Broken

FAULT_01 //
Email as Mandatory Gate
In Tier 2 and 3 markets, email penetration is significantly lower than smartphone penetration. Forcing email-based login excluded a massive, underserved user base before they ever experienced the product.
FAULT_02 //
Social Login Complexity
Google and Apple sign-in options introduced their own confusion — account linking failures, mismatched emails, and app permission dialogs created unexpected drop-off even among users who attempted social login.
FAULT_03 //
Duplicate Account Chaos
Multiple email IDs linked to the same phone number created a fragmented account ecosystem. Users encountered verification loops and confusing state management that made login feel unreliable and broken.

04 // EXECUTION_LOG

Design Process

PHASE_01 //
Research & Data Analysis
Mapped the full login funnel using analytics to identify exact drop-off points. Conducted field interviews in Tier 2/3 cities to understand the relationship users had with their phone numbers vs. email addresses. The insight was clear: phone was identity, email was not.
PHASE_02 //
System Architecture
Designed the logic for handling duplicate accounts — multiple email IDs linked to the same phone number. The "Keep one, sweep the rest" resolution flow was a critical edge case that required clean UX writing and a transparent data-handling approach to maintain user trust.
PHASE_03 //
UI Design & Prototyping
Designed the phone number entry screen with country code selection, OTP verification, and all the associated edge states — expired OTPs, poor network conditions, success confirmations. Each state was prototyped and tested for friction.
PHASE_04 //
Phased Rollout
Phone-based login was strategically positioned as the primary authentication method — not an alternative. The email option remained as a secondary path for existing users. Phased release allowed monitoring of adoption curves without disrupting the existing user base.
05 // BEFORE_AFTER

Before / After

STATE_BEFORE //
The old auth flow
  • Email-only as primary login — no phone option at entry
  • Social login as secondary path — account linking failures common
  • No OTP-based verification — password-dependent
  • Duplicate accounts created silently — users unaware
  • No duplicate resolution flow for phone-linked emails
  • Generic error states with no user-friendly guidance
STATE_AFTER //
The redesigned auth
  • Phone number as primary login — email demoted to secondary
  • 5-digit OTP sent to phone — instant, no password required
  • Country code selector with search — supports global users
  • "Keep one, sweep the rest" — transparent duplicate resolution
  • Retry timer with clear countdown — reduces user confusion
  • Contextual success states — "Verification successful" banner on dashboard
