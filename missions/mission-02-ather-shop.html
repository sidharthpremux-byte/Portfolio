01 // MISSION_BRIEF

Context

Ather Shop's desktop platform performed well, but the mobile experience — serving 85.9% of total traffic — was broken. Clunky navigation, absent search functionality, and a product detail page flow that hadn't been optimised for touch resulted in a catastrophically low 0.8% conversion rate.

The mission: close the performance gap between mobile and desktop, making the smartphone experience the driver of sales growth rather than the primary source of abandonment.

02 // WHAT_WE_NOTICED

Key Findings

INSIGHT_01 //
No Search Capability
The search option was entirely absent from the shopping page, completely hindering users who arrived with a specific product in mind. Intent-driven traffic had no direct path to conversion.
INSIGHT_02 //
Single-Item Purchase Majority
Only 2.5% of users completed multi-item purchases. The vast majority purchased single items, suggesting cart abandonment was happening due to UX friction rather than low purchase intent.
INSIGHT_03 //
Navigation & Filter Issues
Category navigation, filters, and product sorting had usability issues that caused users to lose context mid-browse. The mobile UI didn't carry the same clarity as the desktop — it was a port, not a design.

03 // EXECUTION_LOG

Design Process

PHASE_01 //
Research & Problem Discovery
Analysed session recordings and funnel data to map exactly where mobile users were abandoning. Interviewed Ather Shop users to understand their mental models for product discovery — most arrived with a category in mind, not a specific product name.
PHASE_02 //
Information Architecture
Restructured the navigation hierarchy for touch-first use — clear category entry points, persistent breadcrumb navigation, and a global search bar added to the top bar. The product listing page filter system was redesigned as scrollable chip selectors compatible with one-thumb operation.
PHASE_03 //
Product Detail Page Redesign
The PDP received the most significant intervention: persistent sticky "Buy Now" CTA, horizontal image swipe gallery, inline size guidance with a "how to measure" expandable, and a clear discount display. The add-to-cart and buy-now actions were separated into distinct visual weight classes to reduce decision paralysis.
PHASE_04 //
Phased Release
Released as a phased rollout — search and navigation improvements shipped in Phase 1 to establish a data baseline. The full PDP redesign followed in Phase 2, allowing clean attribution of metric changes to specific design interventions.
04 // BEFORE_AFTER

Before / After

STATE_BEFORE //
The old mobile shop
  • No search bar — zero path for intent-driven product discovery
  • Category browse-only — required navigating full product listings
  • Non-sticky CTA — "Buy Now" scrolled out of view on long PDPs
  • No size guide — users had to guess or leave to find guidance
  • Basic image gallery — no swipe support, arrows only
  • Filter chips required multiple taps — friction-heavy selections
STATE_AFTER //
The redesigned shop
  • Global search with category-scoped results and empty state guidance
  • Category tiles with product photography — visual-first navigation
  • Sticky bottom bar: price + "Add to cart" + "Buy now" always visible
  • Inline size chart with step-by-step measurement guide
  • Full-width swipeable image gallery — touch-native interaction
  • Horizontal scrolling filter chips — one-hand operation optimised
