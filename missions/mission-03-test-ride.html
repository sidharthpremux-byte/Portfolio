01 // MISSION_BRIEF

Context

This mission covers the complete redesign of Ather Energy's online test ride booking system. The core intervention was the introduction of Home Test Rides (HTR) — a first-of-its-kind feature that brought the test ride experience directly to the customer's location, removing the friction of visiting an Experience Centre.

The existing booking flow presented a fragmented, multi-step journey that required users to locate nearby ECs, navigate an unintuitive calendar, and commit personal details before understanding availability — a sequence that drove high abandonment.

02 // PROBLEM_STATEMENT

What Was Broken

FAULT_01 //
Complex Booking Flow
The existing booking sequence was lengthy and cognitively demanding. Users faced multiple decision points with insufficient guidance, leading to confusion and early abandonment before they could commit to a ride.
FAULT_02 //
Ambiguous Location Data
Location information for Experience Centres was presented in a way that created confusion — distances were unclear, and the interface failed to surface the nearest available slot relative to the user's position.
FAULT_03 //
Mandatory EC Visits
Requiring in-person visits to Experience Centres was a structural barrier. The absence of a home test ride option eliminated a major segment of potential customers from completing the flow.

04 // EXECUTION_LOG

Design Process

PHASE_01 //
Research
Conducted user interviews and analysed website analytics to identify key pain points. Focused on where in the funnel abandonment was occurring — mapping emotional friction alongside functional breakdowns.
PHASE_02 //
Iteration
Developed wireframes and interactive prototypes focused on two parallel goals: simplifying the existing EC booking flow, and designing the entirely new HTR path from scratch. Multiple flow variations tested against each other.
PHASE_03 //
Cross-Functional Approach
Worked closely with the development and HTR logistics teams to ensure seamless backend integration. Coordination shaped key decisions around date availability, service radius, and confirmation states.
PHASE_04 //
User Testing
Conducted user testing with 15 participants across both the redesigned EC flow and the new HTR flow. Iterated based on feedback — particularly around location permission handling, date selection clarity, and the booking summary screen.
PHASE_05 //
Phased Release
EC flow improvements shipped first, establishing a baseline, followed by the HTR pilot in selected cities. This staged approach allowed both teams to monitor impact incrementally.
05 // BEFORE_AFTER

Before / After

STATE_BEFORE //
The old booking flow
  • Single-page scroll form — all fields visible at once, overwhelming on mobile
  • No home test ride option — mandatory Experience Centre visit
  • Location selection forced before model selection
  • Ambiguous EC distance display — no proximity sorting
  • Date and time selection buried below fold
  • No booking summary before final confirmation
STATE_AFTER //
The redesigned experience
  • Stepped flow: Location → Date → Time Slot → Details — one decision at a time
  • HTR as primary option: "Test ride right here" surfaces immediately
  • Model selection moved to entry point — users commit to vehicle first
  • ECs sorted by proximity with distance labels
  • Date/time as scannable chips — full week visible at a glance
  • Full booking summary before "Let's go" — reduces cancellations
